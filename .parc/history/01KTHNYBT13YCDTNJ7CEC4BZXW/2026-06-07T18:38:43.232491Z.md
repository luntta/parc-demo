Context: need SEO-friendly marketing pages plus an authenticated app dashboard from one codebase, with good DX.

Decision: Next.js with the App Router. Server components for data-heavy dashboard views, route handlers as a thin BFF in front of the core API. TypeScript end to end.

Consequences: coupling to the Next.js/React release cadence and Vercel-flavoured deployment assumptions. Server/client component boundary is a recurring source of bugs the team must learn.
